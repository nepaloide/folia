# Design: Google OAuth Authentication with JWT

## Technical Approach

Create the first feature module (`modules/auth/`) per `architecture.md` layered pattern. Backend verifies Google ID tokens via `google-auth`, issues 30-day JWTs via `python-jose`, and persists users in PostgreSQL. Mobile uses `expo-auth-session` for Google sign-in and `expo-secure-store` for token persistence. Dev mode fallback preserved when `GOOGLE_CLIENT_ID` is the placeholder value — no regression for local development.

## Architecture Decisions

### Decision: Auth router wiring

| Choice | Alternative | Rationale |
|--------|-------------|-----------|
| Wire `modules/auth/router.py` into existing `api/v1/router.py` | Top-level router in `main.py` | Consistent with existing plants wiring; `/api/v1/auth/google` URL works immediately; avoids parallel routing schemes |

### Decision: Dev mode fallback strategy

| Choice | Alternative | Rationale |
|--------|-------------|-----------|
| If `GOOGLE_CLIENT_ID == "placeholder-local-dev"` AND no JWT → return mock user. If JWT present → verify normally | Always mock / always real in dev | Enables mixed mode: local dev works without creds, but real tokens can be tested against the same instance |

### Decision: User model in module vs flat models/

| Choice | Alternative | Rationale |
|--------|-------------|-----------|
| `modules/auth/models.py` (inside module) | `app/models/user.py` | Architecture.md mandates self-contained modules. Import User into `app/models/__init__.py` for Alembic autodiscovery |

### Decision: Schema vs Model for `get_current_user` return type

| Choice | Alternative | Rationale |
|--------|-------------|-----------|
| Return `UserResponse` Pydantic schema (not SQLAlchemy model) | Return ORM model directly | Each layer owns its contract; router returns schemas, service returns schemas, repo returns models. Existing plants routes return schemas |

## Data Flow

```
                         ┌──────────────────┐
                         │  Google OAuth     │
                         └────────┬─────────┘
                                  │ ID token
                                  ▼
Mobile App ──POST /auth/google──► auth.router
         ◄─── {access_token} ────┤
                                  │
                              auth.service
                                  │
                      ┌───────────┼───────────┐
                      │           │           │
                      ▼           ▼           ▼
                 google-auth  auth.repo  python-jose
                 verify token  upsert     sign JWT
                               user       (sub=uuid)
                                  │
                                  ▼
                              PostgreSQL
```

## Interfaces / Contracts

### API Endpoint

```
POST /api/v1/auth/google
Content-Type: application/json

Request:  { "id_token": "string" }
Response: {
  "access_token": "string",
  "token_type": "bearer",
  "user": {
    "id": "uuid",
    "email": "string",
    "name": "string"
  }
}
Errors:   401 — Invalid token
          400 — Missing id_token
```

### JWT Claims

```json
{
  "sub": "user-uuid-string",
  "exp": 1728000,
  "iat": 1717200000
}
```

### User model (SQLAlchemy)

```python
class User(Base):
    __tablename__ = "users"
    id: Mapped[uuid.UUID] = mapped_column(UUID, primary_key=True, default=uuid.uuid4)
    google_id: Mapped[str] = mapped_column(String, unique=True, nullable=False)
    email: Mapped[str] = mapped_column(String, nullable=False)
    name: Mapped[str] = mapped_column(String, nullable=False)
    timezone: Mapped[str | None] = mapped_column(String, nullable=True)
    notification_hour: Mapped[int | None] = mapped_column(Integer, nullable=True)
    created_at: Mapped[datetime] = mapped_column(TIMESTAMP(timezone=True), server_default=func.now())
```

## Testing Strategy

| Layer | What to Test | Approach |
|-------|-------------|----------|
| Unit | `auth.service.verify_google_token` | Mock `google-auth` return, test valid/invalid/dev-mode paths |
| Unit | `auth.repository` | Async SQLAlchemy session with in-memory SQLite (aiosqlite) |
| Integration | `POST /auth/google` endpoint | FastAPI TestClient, mock service layer, assert JWT in response |
| Integration | Dev mode fallback | TestClient with no auth header, assert mock user returned |
| Integration | Protected endpoint with valid JWT | TestClient with `Authorization: Bearer <signed-jwt>`, assert 200 |
| Unit | Mobile `authStorage.ts` | Mock SecureStore, test token roundtrip |
| Unit | Mobile `useGoogleSignIn.ts` | Mock `expo-auth-session`, test success/error paths |

## Migration / Rollout

**Migration**: `alembic revision --autogenerate -m "add_user_table"` after importing User in `models/__init__.py`. Run with `alembic upgrade head`. Rollback: `alembic downgrade -1`.

No data migration needed — no existing users. No feature flags.
